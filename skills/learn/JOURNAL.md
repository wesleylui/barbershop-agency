# JOURNAL

Central, cross-skill log of lessons captured by the `reflect` skill. Every entry is tagged with the skill it came from so you can filter. Newest at the bottom. Read relevant entries before starting similar work; append after finishing (Step 3).

Format:
```
## YYYY-MM-DD — [skill: <name>] — <subject>
- Task: ...
- Accepted: ...
- Changed/Rejected: ...
- Lesson: ...
```

---

## 2026-06 — [skill: website-builder] — Barbershop (FORM), Calgary
- Task: high-end 3D landing page for a barbershop; later added a floating-reviews section.
- Accepted: rotating 3D barber-pole signature; single self-contained HTML file; conversion-first layout (book CTA above fold, click-to-call, hours, services menu); two-row auto-scrolling reviews marquee with pause-on-hover.
- Changed/Rejected: original brief's "3D photorealistic person" hero (not buildable from code without a model file); Next.js + Tailwind stack (user doesn't code — a double-click file is a better deliverable).
- Lesson: for local service businesses, argue conversion over spectacle and use 3D as an accent; never accept a coded photoreal human; default to single-file HTML for non-coders; flag invented reviews as placeholders to swap.

## 2026-06 — [skill: reflect] — building this skill
- Task: design a reflection skill that captures lessons from any prior skill.
- Accepted: run-at-B / read-back-to-A model; route lessons to both a central journal and the used skill's own LEARNINGS.md; be explicit about the persistence constraint.
- Changed/Rejected: the initial idea of a skill that passively watches other skills run in the background — not possible; nothing executes between messages.
- Lesson: a "learning" skill is really a read-then-append log; framing matters — never imply background watching or silent self-rewriting. Cross-session memory exists only through the saved log that skills read on startup.
